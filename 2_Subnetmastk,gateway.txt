2. SubnetMask = 기준 = 피아식별 = 너네가족or우리가족
    *2진수, 32bit로 구성, 연속된 1과 연속된 0으로 이루어져 있다. ex)11111111.0000000000...
 => 같은 대역대 = 통신 O
    다른 대역대 = 통신 X
 Why? 대역대(=도메인)을 구분하기 위해서!								| *도메인 : 영역, 땅  
     => 같은 대역대는 고객의 요구를 맞추거나, 같은 대역대의 IP들을 부여해주기 위해

 -서브넷 마스크 구성 : Network자리 + HOST자리							|문제1) 192.168.10.1/24 C cl
	  	  1(고정)       0(자유)	 						|   2) 192.168.10.2/24 C cl
  1) A class subnetmask : 255.0.0.0							|   3) 192.168.20.2/24 C cl (*1,2와 통신X)	
  2) B      " 		: 255.255.0.0							|   4) 192.168.30.4/16 B cl (*태생은 C지만 B취급/1,2,3과 통신X)
  3) C      "		: 255.255.255.0							|   5) 192.168.40.5/8  C cl (*앞의 1옥텟이 4번과 같기 때문에 통신O)
 *4) D class : 장비 연결용
 *5) E class : 고갈 대비 및 연구용						
							
**Prefix표기법 = A class:/8 ,B class:/16, C class:/24
ex) C class : 192.168.10.0/24

3. Scope = 대역대
 -1.1.1.1 'A' class /S.M 255.0.0.0/8
 *1.0.0.0 = 대표 주소, Network Address
 *1.255.255.255 = BroadCast Address
 실제 사용 가능 주소범위 : 1.0.0.1 ~ 1.255.255.254 (=2^24-2)
 
 -128.10.1 'B' class / S.M 255.255.0.0/16
 *N.A : 128.10.0
 *B.A : 128.10.255.255
 128.10.0.1 ~ 128.10.255.254 (=2^16-2)

 -192.168.1.1 'C' class / S.M 255.255.255.0/24
 *N.A : 192.168.1.0
 *B.A : 192.168.255.254
 192.168.1.1 ~ 192.168.1.255 (=2^8-2)
-------------------------------------------------------------------------------------------
4. Gateway = 다른 대역대로 가기 위한 관문
-같은 대역대 내에 존재하며 같은 대역대에 있는 컴퓨터들은 같은 G.W IP를 공유
ex) 192.155.10.1/24  G.W 192.155.10.254/24
-대역대 내에서 어떤 IP를 쓰든 상관없으나 끝IP의 앞이 관례
